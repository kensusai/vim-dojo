# デザインモック

**確定版(実装の仕様)は案C**: `home-c.html` / `practice-c.html`。トーンは「道場×レトロゲーム(マイクラ的ブロック質感 × vim ターミナル世界観)」。

- `home.html` / `practice.html` — 案A(ダッシュボード調)。**不採用**(オーナー評: 仕事感がある)。経緯として残す。
- `home-b.html` — 案B(道場×レトロ、和風用語全振り)。**不採用**(方向は正解だが振り切り不足、和風用語はやりすぎ)。
- `home-c.html` / `practice-c.html` — **案C(確定)**。ゲームHUD・ピクセルパネル・ハイテンション師範キャラ・ボス付きワールドマップ。和風用語は「帯」と師範の台詞のみに抑制。

デザイントークンは各ファイルの `:root` に定義。実装時は案Cのトークンを正とする(`practice-c.html` のコメント参照)。

判定の用語: 金メダル=「一本!!」、銀=「技あり!」、銅=「有効」(柔道用語。メダル自体の呼称は金/銀/銅のまま)。

Figma に取り込む場合は html.to.design プラグインの Paste code に HTML 全文を貼る(`skills/design-mockup/SKILL.md` 参照)。
