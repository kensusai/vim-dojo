## 選択された開発プリセット

以下のプリセットが `CLAUDE.md` / `AGENTS.md` のルールと `skills/` の手順書に反映されています:

- **クリーンアーキテクチャ** — ドメイン・ユースケース・アダプタ・インフラを分離し、依存は内側にのみ向ける
- **ドキュメント重視** — ドキュメントは変更と一緒に出荷する。例は実行可能に保つ。決定は記録する
- **パフォーマンス重視** — 計測してから最適化する。計算量の罠を避け、ホットパスを速く保つ
- **継続的リファクタリング** — 触れたコードは少しきれいにして返す。小さく検証しながら、挙動変更とは分離する
- **型安全重視** — 不正な状態を表現不能にする。境界でパースする。型の抜け道を使わない

変更するには `npx agent-ready init --force --presets <ids>` を再実行してください(一覧は `npx agent-ready presets`)。

## 開発フェーズ別ワークフロー

このプロジェクトでは、以下のフェーズ順で開発を進めます。各フェーズには手順書(SKILL.md)と Claude Code 用コマンドがあります。

1. **プロジェクト概要** — 何を・誰のために・なぜ作るのかを整理し、AIエージェントの前提知識をつくる(成果物: `docs/project-overview.md`)
2. **ドメイン理解** — 業務用語・登場人物・業務ルールを整理し、ドメイン知識をAIが参照できる形にする(成果物: `docs/domain.md`)
3. **アーキテクチャ設計** — レイヤー構成・依存方向・モジュール分割を決め、ADR運用を始める(成果物: `docs/architecture.md`)
4. **技術選定** — 言語・フレームワーク・DB等の選定理由と不採用案を docs/tech-stack.md に明文化する(成果物: `docs/tech-stack.md`)
5. **DB設計** — エンティティ・リレーション・migration方針を docs/database.md に整理する(成果物: `docs/database.md`)
6. **CI構築** — lint・テスト・ビルドを自動実行するCIパイプラインを構築し、方針を docs/ci.md に整理する(成果物: `docs/ci.md`)
7. **バックエンド開発** — API設計・エラーハンドリング・認証認可などバックエンド実装ルールに沿って開発する(成果物: `docs/backend.md`)
8. **フロントエンド開発** — 画面設計・コンポーネント・state管理・a11yなどフロントエンド実装ルールに沿って開発する(成果物: `docs/frontend.md`)
9. **インフラ構築** — IaC(CloudFormation / CDK / Terraform 等)でインフラを構築し、方針を docs/infrastructure.md に整理する(成果物: `docs/infrastructure.md`)
10. **監視設計** — ログ・メトリクス・アラート・障害対応手順を docs/monitoring.md に整理する(成果物: `docs/monitoring.md`)

## 推奨ワークフロー

1. タスクの種類に合った `skills/` の手順書を選ぶ(バグ修正・機能追加・レビュー・リファクタリング・テスト・ドキュメント)。
2. エージェントに編集の前に計画させ、曖昧な点には仮定を明示させる。
3. タスクは小さく焦点を絞る — 一度に1つのバグ、1つの機能、1つのリファクタリング。
4. エージェントの差分も他のコントリビューターのPRと同様にレビューする。

## 最新の状態に保つ

- 選択内容(エージェント・プリセット・ワークフロー)は `agent-ready.config.json` に保存されています。同じ選択での再生成は `npx agent-ready sync` だけで行えます。
- `npx agent-ready check` で、このドキュメント群とリポジトリの実態(コマンド・スタック・ファイル欠落)の乖離を検出できます。乖離があると exit 1 になるため CI にも組み込めます。
- `npx agent-ready update` は、手元の編集を保ったまま、古くなったコマンド等の値だけを修復し、欠落ファイルを再生成します。
- `--force` **なし**で `init` を再実行しても既存ファイルは上書きされません。生成ファイルは自由に手で編集できます。
- README の `agent-ready:begin` / `agent-ready:end` マーカー間のセクションは agent-ready が管理します。この中の編集は再生成時に置き換えられます。
- `CLAUDE.md` ⇄ `AGENTS.md` の相互変換は `npx agent-ready convert` でできます。
