最終更新: 2026-07-11(PLAN M1 スパイクの結果)

検証方法: `src/vim/codeMirrorVimEngine.test.ts` によるヘッドレス駆動(`Vim.handleKey` / `Vim.handleEx`)。**このテストとこの表は同期して更新すること。**

| コマンド                  | 判定 | 備考                                                                          |
| ------------------------- | ---- | ----------------------------------------------------------------------------- |

| `j` `k`(上下)             | 🖥    | 上下移動は DOM レイアウト計測に依存し jsdom で駆動不可。M5 の実画面で確認する |

| `i` `a` `o` + `Esc`       | ✅   | モード遷移イベントも検証済み                                                  |

## ステージ3: 検索・置換・グローバルコマンド

| コマンド        | 判定 | 備考                                                                                         |
| --------------- | ---- | -------------------------------------------------------------------------------------------- |
| `/` `?` `n` `N` | 🖥    | 検索は vim 拡張のダイアログ(実 DOM input)経由のためヘッドレス駆動不可。M5 の実画面で確認する |

## エンジンポートの検証済み性質

- `reset()` 後は undo 履歴が前のお題に届かない(お題間の `u` 事故防止)
- キーストローク購読は Esc・モード切替キーも数える(R2)
- バッファ変更購読がクリア判定(R1)に使える粒度で発火する
- レジスタは reset をまたいで残る(本物の vim と同じ挙動として許容)
