# 0008: 進捗の任意クラウドバックアップ(GitHub Gist)

- ステータス: 採用
- 日付: 2026-07-16

## コンテキスト(なぜ判断が必要か)

ADR-0001 でバックエンドレス・ブラウザローカル永続化を採用したが、運用で実害が出た: **ブラウザ(特に Safari/iOS の ITP)は、サイトを約7日間開かないと IndexedDB を自動削除する。** オーナーがしばらく遊ばなかった結果、進捗が初期化された(レッスンクリア・XP がすべて消え、ステージ1に戻った)。エクスポート/インポート(UC6)は用意していたが、手動のため機能していなかった。ローカル完結を維持しつつ、進捗を確実に残す手段が要る。

## 決定(何をどうするか)

- **任意(オプトイン)の GitHub Gist バックアップ**を追加する。アプリはバックエンドを持たず、ブラウザから直接 GitHub API を叩く(ADR-0001 の「サーバーを持たない」は維持)。
- 保存先は**オーナー自身の private gist** 1つ(ファイル名 `vim-dojo-progress.json`、内容はエクスポート JSON と同一)。
- 認証は **`gist` スコープのみの Personal Access Token**。トークンはこの端末の localStorage にのみ保存し、GitHub 以外には送信しない。UI で明示的に有効化する。
- **自動バックアップ**: 進捗変更が落ち着いた後(4秒デバウンス)にエクスポート JSON を gist へ push。失敗はベストエフォート(ローカル保存は成功済みなので警告ログのみ)。
- **自動復元**: 起動時にローカル進捗が空(=ブラウザに消された)かつバックアップ設定済みなら、gist から pull して import してから画面を出す。
- 実装は `src/backup/`(shell 層)。core は関与しない。

## 検討した代替案(採用しなかった案と理由)

- **手動エクスポート/インポートのまま(現状)**: 実際に忘れて被害が出たので不十分。
- **`navigator.storage.persist()` で永続化を要求**: 数行で済むが、許可はブラウザ任せで Safari では効かないことがあり確実でない。補助として併用余地はあるが単独では不足。
- **専用バックエンド/BaaS でアカウント同期**: 最も確実だが ADR-0001(サーバーなし・ローカル完結・コストゼロ)を根本から覆す。単一ユーザーには過剰。
- **リポジトリのファイルに commit**: gist より権限が広く(repo スコープ)、履歴も汚れる。gist が最小。

## 影響(この決定で生じるトレードオフ)

- (+) ブラウザがデータを消しても、次回起動時に自動復元される。複数端末間の緩い移行にも使える。
- (+) 追加コストゼロ(GitHub 無料)、サーバーレスのまま。
- (−) **トークンがブラウザ(localStorage)に平文で載る。** スコープを gist 限定にして影響範囲を最小化するが、端末が侵害されれば gist は読める。個人利用・自分の端末という前提で受容する。UI で注意喚起する。
- (−) オフライン時・API 障害時はバックアップされない(ベストエフォート)。ローカル保存は常に先に成功しているので、体験は劣化しない。
- (−) gist の内容は「vim の練習進捗」のみで機微情報を含まない(domain.md 禁止事項: 進捗以外を送らない)。
