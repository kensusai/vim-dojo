# テスト作成

## いつ使うか

コードのカバレッジが不足しているとき、または変更にリグレッション保護が必要なとき。

## 手順

1. 対象コードを読み、実装の詳細ではなく観測可能な挙動を特定する。
2. ケースを列挙する: 正常系、境界値、異常系。
3. 既存のテスト規約(構造・命名・ヘルパー)に従う。
4. 挙動が壊れたときに失敗するテストを書く。不安なら一時的にコードを壊して失敗を確認する。

## ルール

- 公開インターフェース経由で挙動をテストする。直接呼べるものをモックしない。
- 正確にアサートできるロジックにスナップショットテストを使わない。
- 決して失敗しないテストはテストが無いより悪い。すべてのテストは何かを守っていること。
