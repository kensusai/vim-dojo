# バグ修正

## いつ使うか

不具合が報告されたとき: 誤った挙動、クラッシュ、リグレッション。

## 手順

1. **再現する。** バグが実際に発生することを確認し、発生条件を正確に把握する。
2. **テストで捉える。** テストコマンドが未検出のため、確実な手動再現手順を確立して記録する。
3. **根本原因を特定する。** 症状が出た場所ではなく、失敗の発生源まで遡る。
4. **最小限に修正する。** 根本原因に対処する最小の変更を行う。
5. **検証する。** 手動再現を再実行し、バグが消えたことを確認する。
6. **報告する。** 根本原因・修正内容・残りのリスクやフォローアップを要約する。

## ルール

- バグの範囲を超えて変更を広げない。無関係な改善は別途提案する。
- 再現できない場合は、推測で修正せず、その事実を報告する。
- 修正が他の箇所の挙動変更を必要とする場合は、明示的に告知する。
