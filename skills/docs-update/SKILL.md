# ドキュメント更新

## いつ使うか

挙動・コマンド・セットアップが変わったとき。またはドキュメントが誤っている・欠けているとき。

## 手順

1. 何が変わったかを特定し、それに言及するすべてのドキュメントを見つける: README、`docs/`、`CLAUDE.md`、`AGENTS.md`、コードコメント。
2. すべての記述を実際のコードと照合する。「あるべき姿」ではなく「実際の姿」を文書化する。
3. コピー&ペーストで実行できるコマンドとともにドキュメントを更新する。
4. 相互参照とリンクが依然として有効か確認する。

## ルール

- 未検証の挙動を文書化しない。
- 生成されたセクション(`agent-ready:begin` / `agent-ready:end` マーカー間)は手で編集せず、再生成に任せる。
- 新しいドキュメントを作るより、既存ドキュメントの更新を優先する。

## ドキュメントファーストの規律

- どのタスク中でも、見つけた欠落・陳腐化したドキュメントはそのタスクのスコープの一部として扱う。任意のフォローアップにしない。
- 使い方だけでなく、設計判断 — 何を選び、何を退け、なぜか — を記録する。
