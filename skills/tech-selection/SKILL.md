# フェーズ 5: 技術選定

## 目的

「なぜこの技術なのか」を `docs/tech-stack.md` に明文化する。AIエージェントが勝手に別ライブラリを持ち込んだり、選定済みの技術を迂回したりすることを防ぐ。

## 進め方

1. 現在のスタック(Unknown stack)を出発点に、リポジトリの依存関係と設定ファイルから採用技術を棚卸しする。
2. 各カテゴリについて、採用技術・採用理由・検討した代替案を `docs/tech-stack.md` に記録する。
3. 主要な選定(全体スタック、DB、認証方式など)は `docs/adr/0001-tech-stack.md` のように ADR にも残す。
4. 未決定のカテゴリは「未選定」と明記し、選定基準と候補を書く。

## docs/tech-stack.md に含めるカテゴリ

- 言語 / ランタイム
- フレームワーク
- DB / ストレージ
- ORM / データアクセス
- 認証・認可
- インフラ / デプロイ先
- テスト(ユニット / E2E)
- 監視 / ログ基盤
- その他主要ライブラリ

各カテゴリに: **採用技術 / 採用理由 / 採用しなかった候補と理由 / 見直し条件**

## ルール

- 新しい依存を追加する前に、このドキュメントの方針と照合する。方針にない依存は追加前に提案として挙げる。
- 「流行っているから」は採用理由にしない。要件・制約に紐づけて書く。
