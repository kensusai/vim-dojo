# フェーズ 1: プロジェクト概要

## 目的

AIエージェントが作業を始める前に最初に理解すべき情報を、`docs/project-overview.md` として整理する。以降のすべてのフェーズと実装判断の土台になる。

## 進め方

1. README・既存ドキュメント・イシュー・コードを読み、現時点で分かっている情報を集める。
2. 以下の項目を埋めた `docs/project-overview.md` を作成(または更新)する。分からない項目は「未確定」と明記し、勝手に推測で埋めない。
3. 質問リストを作り、未確定事項をユーザー(プロダクトオーナー)に確認する。
4. 確定した内容を README の冒頭と `CLAUDE.md` / `AGENTS.md` のプロジェクト概要に反映する。

## docs/project-overview.md に含める内容

- **何を作るのか** — プロダクトの一言説明と少し詳しい説明
- **誰のためのものか** — 対象ユーザー・ペルソナ
- **解決する課題** — 現状の問題と、このプロダクトによる解決
- **主要機能** — 優先順位付きの機能一覧(やらないことも明記)
- **非機能要件** — 性能・可用性・セキュリティ・スケール想定
- **成功条件** — 何をもって成功とするか(計測可能な指標)
- **制約条件** — 期限・予算・技術・法令・組織上の制約

## ルール

- 事実(確認済み)と仮説(未確認)を必ず区別して書く。
- このドキュメントと矛盾する実装依頼を受けたら、着手前に矛盾を指摘する。
